Exact Expected Runtime of Quicksort

Comparisons

Lemma 1: The running time of Quicksort on an nn-element array is O(n+X)O(n+X), where XX is the number of element comparisons performed.

Proof: Each time partition is called, the selected pivot is never included in any future calls to quicksort or partition. Thus, there will be at most nn calls to Partition over the entire execution.

Partition takes O(1)O(1) time plus an amount of time proportional to the number of iterations of the for loop. Each iteration performs one element comparison. There are at most nn calls to Partition.

Thus, the total time for quicksort is O(n+X)O(n+X).

Lemma 2: During the execution of Randomized-Quicksort on an array of nn distinct elements z1<z2<…,zjz_1 < z_2 < \ldots, z_j, an element ziz_i is compared with an element zjz_j, where i<ji<j, if and only if one of them is chosen as a pivot before any other element in the set ZijZ_{ij}. No two elements are ever compared twice.

Proof: Consider the case where it is the first time an element x∈Zijx \in Z_{ij} is chosen as a pivot. There are three cases to consider:

  • If xx is neither ziz_i nor zjz_j, then ziz_i and zjz_j are not compared at any subsequent time, because they will be partitioned by xx.
  • If x=zix=z_i, then Partition compares ziz_i with every other item in ZijZ_{ij}. Similarly, if x=zjx=z_j, then Partition compares zjz_j with every other item in ZijZ_{ij}. Thus, ziz_i and zjz_j are compared if and only if one of them is the first to be chosen as a pivot out of ZijZ_{ij}. Since the pivot is removed from future comparisons, it is never compared again with any other element, thus each pairwise comparison of elements only occurs at most once.

Lemma 3: Consider an execution of Randomized-Quicksort on an array of nn distinct elements z1<z2<…<znz_1 < z_2 < \ldots < z_n. Given two arbitrary elements ziz_i and zjz_j where i<ji<j, the probability that they are compared is 2/(j−i+1)2/(j-i+1).

Proof: Consider the tree of recursive calls made by Randomized-Quicksort, and the set of elements provided as input to each call. Initially, the root set contains all elements of ZijZ_{ij}. These elements stay together (and are not compared to each other) until Partition chooses some x∈Zijx \in Z_{ij} as the pivot. From that point on, the pivot xx appears in no subsequent input and is not compared.

The first time that we choose a pivot x∈Zijx \in Z_{ij}, each element is equally likely. Since |Zij|=j−i+1|Z_{ij}| = j-i+1, the probability is 1/(j−i+1)1/(j-i+1) that any given element is chosen as the pivot. Thus, by Lemma 2, we have:

$$ \begin{align\*} Pr(z_i \text{ is compared to } z_j) = & Pr(z_i \text{or} z_j \text{first pivot}) \\ = & Pr(z_i \text{first pivot}) + Pr(z_j \text{first pivot}) \\ = & \frac{2}{j-i+1} \end{align\*} $$

where the second line follows from the first because the two events are mutually exclusive.

Theorem: The expected running time of Randomized-Quicksort on nn distinct elements is O(nlg⁡n)O(n \lg{n}).

Proof: Let the elements be z1<z2<…<znz_1 < z_2 < \ldots < z_n. Define an indicator variable $X_{ij} = [z_i \text{ is compared to } z_j\]$. From Lemma 2, each pair is compared at most once, so we can express XX as

X=∑i=1n−1∑j=i+1nXij X = \sum_{i=1}^{n-1} \sum_{j=i+1}^n X_{ij}

By taking expectations of both sides, and by linearity of expectation, we obtain:

$$ \begin{align\*} E\[X\] = & E \large[ \sum_{i=1}^{n-1} \sum_{j=i+1}^n X_{ij} \large] \\ = & \sum_{i=1}^{n-1} \sum_{j=i+1}^n E\[ X_{ij} ] \\ = & \sum_{i=1}^{n-1} \sum_{j=i+1}^n Pr(z_i \text{compared to} z_j) \\ = & \sum_{i=1}^{n-1} \sum_{j=i+1}^n \frac{2}{j-i+1} \end{align\*} $$

Change of variables:

$$ \begin{align\*} E\[X\] = & \sum_{i=1}^{n-1} \sum_{k=1}^{n-i} \frac{2}{k+1} \\ < & \sum_{i=1}^{n-1} \sum_{k=1}^{n} \frac{2}{k} \\ = & \sum_{i=1}^{n-1} O(\lg{n}) \\ = & O(n\lg{n}) \end{align\*} $$
